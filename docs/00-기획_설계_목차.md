# VHDL 파싱 및 시각적 디버깅 — 기획·설계 문서 목차

본격적인 코딩에 앞서 기획과 설계를 문서화하여, 복잡한 파서 개발 시 방향을 잃지 않도록 합니다.  
향후 **웹 기반 테스트 환경 및 VS Code 확장 프로그램(Extension)**과의 연동을 고려한 구조로 작성되었으며, **4단계 AST 심층 파싱** 및 **Tree-sitter 엔진**(점진적 파싱·WASM 경량화 대비) 도입 방향을 반영합니다.

## 활용 방법

- **구현 전:** PSD로 전체 구조를 잡고, PRD로 우선순위(필수/높음)를 확인한 뒤, 기능 상세 명세서대로 입·출력 계약을 맞춘다.
- **구현 중:** 각 모듈 단위로 명세의 Input/Output/Process를 기준으로 단위 테스트를 작성한다.
- **연동 시:** JSON Exporter 출력 스키마(및 AST 전체 덤프)를 **웹 프론트엔드 및 VS Code Webview**와 공유하여 API/파일 계약을 유지한다.
- **검증:** 문법별 마이크로 테스트 파일(`test_*.vhd`)과 AST Full Dump로 누락 문법이 없는지 확인한다.
